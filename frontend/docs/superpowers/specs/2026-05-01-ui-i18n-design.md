# UI 다국어(i18n) 설계

## 지원 언어 (7개)
`en`, `ko`, `ja`, `zh-Hans`, `ru`, `mn`, `vi`

## 언어 결정 우선순위
1. 로그인 상태 → 서버 `user.preferredLanguage`
2. 비로그인 → 기기 언어 (`expo-localization`) → 지원 안 하면 `en` 폴백
3. 언제든 설정에서 수동 변경 가능 → 서버 PATCH + AsyncStorage 동기화

## 아키텍처
- **i18next + react-i18next** — 정적 UI 문자열 번역
- **번역 파일 위치:** `locales/{en,ko,ja,zh-Hans,ru,mn,vi}.json`
- **언어 캐시:** AsyncStorage `appLanguage` 키 — 오프라인/재시작 대응

## 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `i18n/index.ts` | i18next 초기화 (신규) |
| `locales/*.json` | 7개 언어 번역 파일 (신규) |
| `app/_layout.tsx` | 앱 시작 시 i18next init |
| `stores/authStore.ts` | 로그인/로드 시 preferredLanguage → i18next 언어 변경 |
| `app/(auth)/register.tsx` | 국가 선택 → language code 매핑 → preferredLanguage 서버 저장 |
| `app/account-settings.tsx` | 언어 변경 UI 추가 |
| `app/language-setup.tsx` | 삭제 (기기 언어 자동 감지로 대체) |

## 국가 → 언어 매핑
- `KR` → `ko`
- `JP` → `ja`
- `CN`, `TW`, `HK` → `zh-Hans`
- `RU`, `KZ`, `KG`, `TJ`, `TM`, `UZ` → `ru` (중앙아시아 키릴 계열 → 러시아어)
- `MN` → `mn`
- `VN`, `LA`, `KH` → `vi`
- 나머지 → `en`

## 서비스 API
- `PATCH /users/fcm-token` 옆에 `PATCH /users/language` 또는 기존 프로필 수정 API 활용 (백엔드 확인 필요)
