App Overview

요청, 매칭, 소통, 리뷰까지 하나의 흐름으로.

단순 번역 앱이 아니라 실제 상황을 해결하기 위한 생활 밀착형 도움 매칭 서비스입니다. 유학생은 도움을 요청하고, 헬퍼는 요청을 확인한 뒤 채팅과 영상통화로 세부 내용을 조율합니다.

Problem

유학생이 혼자 해결하기 어려운 순간을 줄입니다.

한국 생활에서 가장 어려운 건 정보가 없는 것이 아니라, 실제 상황에서 누구에게 어떻게 도움을 요청해야 하는지 모르는 순간입니다.

1

언어 장벽

병원, 관공서, 학교 행정 상황에서 한국어 표현이 어려워 정확한 의사소통이 힘들 수 있습니다.

2

생활 정보 부족

학교 공지, 식단, 지역 정보, 생활 절차를 여러 곳에서 따로 찾아야 하는 불편함이 있습니다.

3

신뢰 문제

무작정 오픈채팅이나 커뮤니티에 의존하면 도움을 받을 사람을 신뢰하기 어렵습니다.

Features

앱 서비스에 필요한 핵심 기능을 하나로 묶었습니다.

Kutoring은 도움 요청, 헬퍼 추천, 실시간 소통, 커뮤니티, 리뷰와 신고까지 실제 서비스 운영에 필요한 기능을 포함합니다.

01
회원가입 / 로그인
이메일 인증과 JWT 기반 인증으로 사용자를 안전하게 구분하고 관리합니다.
Core
02
도움 요청 매칭
병원 동행, 행정 처리, 통역, 생활 도움 등 상황별 요청을 등록하고 헬퍼와 연결됩니다.
Matching
03
AI 헬퍼 추천
요청자의 상황과 도움 카테고리를 바탕으로 적합한 헬퍼를 추천하는 흐름을 제공합니다.
AI
04
실시간 채팅
매칭 이후 요청자와 헬퍼가 WebSocket 기반 채팅으로 세부 내용을 조율합니다.
Realtime
05
영상 통화
직접 만나기 전이나 원격 도움이 필요한 경우 영상 통화를 통해 상황을 확인할 수 있습니다.
Call
06
커뮤니티 / 공지 / 식단
유학생이 필요한 학교생활 정보, 커뮤니티 게시글, 공지사항, 식단 정보를 확인할 수 있습니다.
Info
07
리뷰 / 신고 / 관리자
도움 이후 리뷰를 남기고, 문제가 있는 사용자는 신고하며, 관리자는 운영 상태를 관리합니다.
Trust
Flow

사용자는 복잡한 절차 없이 도움을 요청합니다.

앱 소개 페이지에서는 기능을 길게 설명하는 것보다, 사용자가 어떤 순서로 문제를 해결하는지 보여주는 게 더 중요합니다.

사용자 흐름 4 steps
1

도움 카테고리 선택

병원, 행정, 통역, 생활 도움 중 필요한 상황을 선택합니다.

2

요청 내용 작성

장소, 시간, 필요한 언어, 구체적인 상황을 입력합니다.

3

헬퍼와 연결

요청에 맞는 헬퍼와 연결되고 채팅을 시작합니다.

4

도움 완료 후 리뷰

도움이 끝난 뒤 리뷰를 남겨 신뢰 기반 매칭 환경을 만듭니다.

서비스 구조 system
// Kutoring service architecture

사용자 앱 = React Native + Expo
인증 = Email Verification + JWT
백엔드 = Java Spring Boot REST API
실시간 통신 = WebSocket Chat
AI 서버 = FastAPI 기반 추천/번역 흐름
데이터 = MySQL
배포 = Railway + Docker

// Goal
외국인 유학생의 실제 생활 문제를
요청 → 매칭 → 소통 → 완료 흐름으로 해결
